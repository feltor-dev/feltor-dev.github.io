Detailed Description

template<size_t k, class ContainerType>
struct dg::AB< k, ContainerType >

Struct for Adams-Bashforth explicit multistep time-integration

\[ u^{n+1} = u^n + \Delta t\sum_{j=0}^k b_j f\left(u^{n-j}\right) \]

.

Computes

\[ u_{n+1} = u_n + dt\sum_{j=0}^k b_j f(u_{n-j}) \]

Uses only blas1::axpby routines to integrate one step and only one right-hand-side evaluation per step.

Template Parameters
kOrder of the method (Currently one of 1, 2, 3, 4 or 5)
ContainerTypeAny class for which a specialization of TensorTraits exists and which fulfills the requirements of the there defined data and execution policies derived from AnyVectorTag and AnyPolicyTag. Among others
  • dg::HVec (serial), dg::DVec (cuda / omp), dg::MHVec (mpi + serial) or dg::MDVec (mpi + cuda / omp)
  • std::vector<dg::DVec> (vector of shared device vectors), std::array<double, 4> (array of 4 doubles)
  • double (scalar) and other primitive types ...
If there are several ContainerTypes in the argument list, then TensorTraits must exist for all of them

Initialize first step. Call before using the step function.

This routine initiates the first steps in the multistep method by integrating backwards in time with Euler's method. This routine has to be called before the first timestep is made.

Template Parameters
RHSThe right hand side is a functor type with no return value (subroutine) of signature void operator()(value_type, const ContainerType&, ContainerType&) The first argument is the time, the second is the input vector, which the functor may not override, and the third is the output, i.e. y' = f(t, y) translates to f(t, y, y'). The two ContainerType arguments never alias each other in calls to the functor.
Parameters
rhsThe rhs functor
t0The intital time corresponding to u0
u0The initial value of the integration
dtThe timestep
Note
the implementation is such that on output the last call to the rhs is at (t0,u0). This might be interesting if the call to the rhs changes its state.
