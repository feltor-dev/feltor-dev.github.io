118 /*
119  compared to unpreconditioned compare
120  dot(r,r), axpby()
121  to
122  dot( r,P,r), symv(P)
123  i.e. it will be slower, if P needs to be stored
124  (but in our case P_{ii} can be computed directly
125  compared to normal preconditioned compare
126  ddot(r,P,r), dsymv(P)
127  to
128  ddot(r,z), dsymv(P), axpby(), (storage for z)
129  i.e. it's surely faster if P contains no more elements than z
130  (which is the case for diagonal scaling)
131  NOTE: the same comparison hold for A with the result that A contains
132  significantly more elements than z whence ddot(r,A,r) is far slower than ddot(r,z)
133 */
