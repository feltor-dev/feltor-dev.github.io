107 /*
108  compared to unpreconditioned compare
109  dot(r,r), axpby()
110  to
111  dot( r,P,r), symv(P)
112  i.e. it will be slower, if P needs to be stored
113  (but in our case P_{ii} can be computed directly
114  compared to normal preconditioned compare
115  ddot(r,P,r), dsymv(P)
116  to
117  ddot(r,z), dsymv(P), axpby(), (storage for z)
118  i.e. it's surely faster if P contains no more elements than z
119  (which is the case for diagonal scaling)
120  NOTE: the same comparison hold for A with the result that A contains
121  significantly more elements than z whence ddot(r,A,r) is far slower than ddot(r,z)
122 */
