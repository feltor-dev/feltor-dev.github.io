112 /*
113  compared to unpreconditioned compare
114  dot(r,r), axpby()
115  to
116  dot( r,P,r), symv(P)
117  i.e. it will be slower, if P needs to be stored
118  (but in our case P_{ii} can be computed directly
119  compared to normal preconditioned compare
120  ddot(r,P,r), dsymv(P)
121  to
122  ddot(r,z), dsymv(P), axpby(), (storage for z)
123  i.e. it's surely faster if P contains no more elements than z
124  (which is the case for diagonal scaling)
125  NOTE: the same comparison hold for A with the result that A contains
126  significantly more elements than z whence ddot(r,A,r) is far slower than ddot(r,z)
127 */
