122 /*
123  compared to unpreconditioned compare
124  dot(r,r), axpby()
125  to
126  dot( r,P,r), symv(P)
127  i.e. it will be slower, if P needs to be stored
128  (but in our case P_{ii} can be computed directly
129  compared to normal preconditioned compare
130  ddot(r,P,r), dsymv(P)
131  to
132  ddot(r,z), dsymv(P), axpby(), (storage for z)
133  i.e. it's surely faster if P contains no more elements than z
134  (which is the case for diagonal scaling)
135  NOTE: the same comparison hold for A with the result that A contains
136  significantly more elements than z whence ddot(r,A,r) is far slower than ddot(r,z)
137 */
