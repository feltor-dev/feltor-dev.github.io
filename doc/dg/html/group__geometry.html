Detailed Description

These routines form the heart of our geometry free numerical algorithms. They are called by our geometric operators like the Poisson bracket.

Multiply a tensor with a vector in 2d.

Compute \( w^i = t^{ij}v_j\) for \( i,j\in \{1,2\}\) in the first two dimensions (ignores the 3rd dimension in t)

  • dg::HVec (serial), dg::DVec (cuda / omp), dg::MHVec (mpi + serial) or dg::MDVec (mpi + cuda / omp)
  • std::vector<dg::DVec> (vector of shared device vectors), std::array<double, 4> (array of 4 doubles)
  • double (scalar) and other primitive types ...
If there are several ContainerTypes in the argument list, then TensorTraits must exist for all of them
See also
See The Level 1 dispatch system for a detailed explanation of our type dispatch system
Parameters
tinput Tensor
in0(input) first component (restricted)
in1(input) second component (may alias out1)
out0(output) first component (restricted)
out1(output) second component (may alias in1)
Attention
aliasing only allowed between out1 and in1
Note
Currently required memops:
  • 10(9) reads + 2 writes if all values in t are set; (-1 read if alias is used)
  • 6(5) reads + 2 writes if t is diagonal; (-1 read if alias is used)
  • 4 reads + 2 writes if t is empty and no alias
  • 2 reads + 1 writes (= 1 copy) if t is empty and out1 aliases in1
